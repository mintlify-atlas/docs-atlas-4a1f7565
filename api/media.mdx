## Media items

Seerr tracks media internally as `MediaInfo` objects. These represent movies and TV shows that have been requested or are available in your library.

### MediaInfo object

<ResponseField name="id" type="number">
  Seerr internal media ID.
</ResponseField>

<ResponseField name="tmdbId" type="number">
  TMDB ID.
</ResponseField>

<ResponseField name="tvdbId" type="number">
  TheTVDB ID (TV only, may be null).
</ResponseField>

<ResponseField name="status" type="number">
  Availability status: `1` = Unknown, `2` = Pending, `3` = Processing, `4` = Partially Available, `5` = Available, `6` = Deleted.
</ResponseField>

<ResponseField name="requests" type="array">
  List of `MediaRequest` objects associated with this media item.
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 timestamp of when the item was first tracked.
</ResponseField>

<ResponseField name="updatedAt" type="string">
  ISO 8601 timestamp of the last update.
</ResponseField>

## List media

Returns all media items tracked by Seerr.

**`GET /media`**

<ParamField query="take" type="number">
  Number of results to return.
</ParamField>

<ParamField query="skip" type="number">
  Number of results to skip.
</ParamField>

<ParamField query="filter" type="string">
  Filter by availability: `all`, `available`, `partial`, `allavailable`, `processing`, `pending`, `deleted`.
</ParamField>

<ParamField query="sort" type="string" default="added">
  Sort order: `added`, `modified`, or `mediaAdded`.
</ParamField>

<Tabs>
  <Tab title="curl">
    ```bash
    curl -X GET "http://localhost:5055/api/v1/media?filter=available&take=20" \
      -H "X-Api-Key: your-api-key"
    ```
  </Tab>
  <Tab title="Response">
    ```json
    {
      "pageInfo": {
        "page": 1,
        "pages": 10,
        "results": 200
      },
      "results": [
        {
          "id": 1,
          "tmdbId": 438631,
          "tvdbId": null,
          "status": 5,
          "requests": [],
          "createdAt": "2024-01-01T00:00:00.000Z",
          "updatedAt": "2024-03-01T00:00:00.000Z"
        }
      ]
    }
    ```
  </Tab>
</Tabs>

## Delete a media item

Removes a media item from Seerr. Requires `MANAGE_REQUESTS`.

**`DELETE /media/{mediaId}`**

```bash
curl -X DELETE "http://localhost:5055/api/v1/media/1" \
  -H "X-Api-Key: your-api-key"
```

Returns `204 No Content` on success.

---

## Update media status

Manually sets the availability status of a media item.

**`POST /media/{mediaId}/{status}`**

Path parameter `status` must be one of: `available`, `partial`, `processing`, `pending`, `unknown`, `deleted`.

<ParamField body="is4k" type="boolean">
  When `true`, updates the 4K status field. When `false` or omitted, updates the standard status field.
</ParamField>

<Tabs>
  <Tab title="Mark available">
    ```bash
    curl -X POST "http://localhost:5055/api/v1/media/1/available" \
      -H "X-Api-Key: your-api-key" \
      -H "Content-Type: application/json" \
      -d '{}'
    ```
  </Tab>
  <Tab title="Mark 4K available">
    ```bash
    curl -X POST "http://localhost:5055/api/v1/media/1/available" \
      -H "X-Api-Key: your-api-key" \
      -H "Content-Type: application/json" \
      -d '{"is4k": true}'
    ```
  </Tab>
</Tabs>

Returns the updated `MediaInfo` object.

## Delete a media file

Removes the actual media file from Radarr or Sonarr. Requires `ADMIN`.

**`DELETE /media/{mediaId}/file`**

<ParamField query="is4k" type="boolean">
  When `true`, removes from the 4K service instance.
</ParamField>

```bash
curl -X DELETE "http://localhost:5055/api/v1/media/1/file" \
  -H "X-Api-Key: your-api-key"
```

## Get media watch data

Returns play statistics and a list of users who have watched a media item. Requires `ADMIN`.

**`GET /media/{mediaId}/watch_data`**

```bash
curl -X GET "http://localhost:5055/api/v1/media/1/watch_data" \
  -H "X-Api-Key: your-api-key"
```

```json
{
  "data": {
    "playCount7Days": 3,
    "playCount30Days": 12,
    "playCount": 45,
    "users": [
      { "id": 1, "email": "alice@example.com" }
    ]
  },
  "data4k": {
    "playCount7Days": 0,
    "playCount30Days": 2,
    "playCount": 8,
    "users": []
  }
}
```

## Movie details

Fetches full details for a movie from TMDB.

**`GET /movie/{movieId}`**

<ParamField path="movieId" type="number" required>
  TMDB movie ID.
</ParamField>

<ParamField query="language" type="string">
  ISO 639-1 language code for localized data.
</ParamField>

```bash
curl -X GET "http://localhost:5055/api/v1/movie/438631" \
  -H "X-Api-Key: your-api-key"
```

<AccordionGroup>
  <Accordion title="Response fields">
    <ResponseField name="id" type="number">TMDB movie ID.</ResponseField>
    <ResponseField name="imdbId" type="string">IMDb ID (e.g. `tt1160419`).</ResponseField>
    <ResponseField name="title" type="string">Movie title.</ResponseField>
    <ResponseField name="originalTitle" type="string">Title in original language.</ResponseField>
    <ResponseField name="overview" type="string">Plot synopsis.</ResponseField>
    <ResponseField name="releaseDate" type="string">Release date.</ResponseField>
    <ResponseField name="runtime" type="number">Runtime in minutes.</ResponseField>
    <ResponseField name="budget" type="number">Production budget in USD.</ResponseField>
    <ResponseField name="revenue" type="number">Box office revenue in USD.</ResponseField>
    <ResponseField name="status" type="string">Production status (e.g. `Released`).</ResponseField>
    <ResponseField name="tagline" type="string">Movie tagline.</ResponseField>
    <ResponseField name="voteAverage" type="number">TMDB vote average.</ResponseField>
    <ResponseField name="voteCount" type="number">TMDB vote count.</ResponseField>
    <ResponseField name="popularity" type="number">TMDB popularity score.</ResponseField>
    <ResponseField name="genres" type="array">List of genre objects with `id` and `name`.</ResponseField>
    <ResponseField name="productionCompanies" type="array">List of production company objects.</ResponseField>
    <ResponseField name="credits" type="object">
      Cast and crew.
      <Expandable title="Credits fields">
        <ResponseField name="cast" type="array">List of cast members with `name`, `character`, and `profilePath`.</ResponseField>
        <ResponseField name="crew" type="array">List of crew members with `name`, `job`, `department`, and `profilePath`.</ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="collection" type="object">Movie collection info if part of a franchise.</ResponseField>
    <ResponseField name="externalIds" type="object">External IDs including `imdbId`, `tvdbId`, and social media IDs.</ResponseField>
    <ResponseField name="mediaInfo" type="object">Seerr availability status and requests.</ResponseField>
    <ResponseField name="watchProviders" type="array">Streaming availability by region.</ResponseField>
    <ResponseField name="relatedVideos" type="array">Trailers, teasers, and clips.</ResponseField>
  </Accordion>
</AccordionGroup>
